# M5Dial-Badge 仕様書

作成日: 2026-07-09
対象ハードウェア: M5Stack M5Dial(M5StampS3 / ESP32-S3FN8、Flash 8MB、PSRAM なし)

## 1. 用語

| 用語 | 意味 |
| --- | --- |
| コンテンツ | LCD に表示する 1 単位。静止画(image)または動画(video) |
| プレイリスト | 保存済みコンテンツの表示順リスト |
| スライドショー | プレイリストを自動巡回する基本動作状態 |
| 2 段階 QR | ① Wi-Fi 参加 QR(`WIFI:T:WPA;S:...;P:...;;`)→ ② URL QR(`http://...`)の順で LCD に表示する誘導方式 |

## 2. システム構成

```text
スマホ(標準カメラ + ブラウザ)
   │ ① Wi-Fi QR 読み取り → AP 参加(または同一 LAN)
   │ ② URL QR 読み取り → ブラウザで Web UI を開く
   ▼
M5Dial-Badge
   ├─ Wi-Fi: SoftAP(デフォルト)/ STA(設定時)+ AP フォールバック
   ├─ Web サーバー(ポート 80): Web UI + REST API + キャプティブポータル
   ├─ LittleFS: コンテンツ / プレイリスト / 設定 / Wi-Fi 資格情報
   └─ LCD: スライドショー再生 / QR 表示 / ステータス表示
```

## 3. 機能仕様

### 3.1 Wi-Fi(ハイブリッド)

- **起動シーケンス**
  1. LittleFS に STA 資格情報があれば STA 接続を試行(タイムアウト 15 秒)
  2. 成功 → STA モードで動作。mDNS `badge.local` を開始
  3. 失敗または資格情報なし → SoftAP モードで動作
- **SoftAP**
  - SSID: `M5Dial-Badge`、**パスワードなし(オープン AP)**。Wi-Fi QR を読むだけで 1 タップ接続できることを優先する。電波圏内の第三者が接続してコンテンツを変更できるリスクは、バッヂという用途上許容する(将来必要になれば設定でパスワード有効化を追加)
  - IP: `192.168.4.1`
  - DNS サーバーで全ドメインを自 IP に解決(キャプティブポータル)
  - OS の接続性チェック URL(Android `/generate_204` 系、iOS `/hotspot-detect.html` 系、Windows `/ncsi.txt` 系)には「ブラウザで `http://192.168.4.1` を開いてください」という案内ページを返す。ポータル内疑似ブラウザでのアップロードはサポートしない(不安定なため)
- **STA**
  - Web UI の設定画面から SSID / パスワードを入力して保存(LittleFS)
  - 保存後に再起動して STA 接続。失敗時は SoftAP に自動フォールバックし、その旨を LCD に表示
  - STA 時も Web UI・API は同一。URL QR は取得 IP で動的生成
- **モード表示**: LCD ステータス画面に現在のモード(AP / STA)、SSID、IP を表示

### 3.2 QR 表示(2 段階)

- **QR モード遷移**: 物理ボタン(BtnA)長押し 2 秒で QR モードへ
- **AP モード時**
  - 1 画面目: Wi-Fi 参加 QR(`WIFI:T:nopass;S:M5Dial-Badge;;`)+「1/2 Wi-Fi に接続」の案内
  - 2 画面目: URL QR(`http://192.168.4.1/`)+「2/2 ブラウザで開く」の案内
  - ダイヤル回転またはタッチで 1 ↔ 2 を切り替え
- **STA モード時**: URL QR(`http://<取得IP>/`)のみ表示。補助として `http://badge.local` も文字表示
- **終了**: BtnA 短押しでスライドショーに戻る
- QR は円形画面の内接正方形(約 170×170px)に収まるサイズで描画する

### 3.3 コンテンツ形式

- **静止画**: 240×240 JPEG。ブラウザ側で中央クロップ + リサイズ + JPEG 化(品質 0.8 目安)して転送。端末は受信データを無変換で保存・表示
- **動画**: 独自 MJPG コンテナ(ブラウザ側で生成)

```text
MJPG コンテナ(リトルエンディアン):
  magic   : "MJPG"(4 バイト)
  version : u8 = 1
  fps     : u8(デフォルト 10)
  frames  : u16(フレーム数)
  reserved: u32
  以降フレームの繰り返し: [u32 size][JPEG データ(240×240)]
```

- ブラウザ側変換仕様(Web UI)
  - `<video>` + `<canvas>` で任意の動画ファイル(MP4 / MOV / WebM 等ブラウザが再生できるもの)からフレームをキャプチャ
  - fps はデフォルト 10(UI で 5 / 10 / 15 を選択可)、最大長は残容量から自動算出して UI に表示、超過分は切り捨て(開始位置は UI で選択)
  - JPEG 品質はデフォルト 0.7、UI で調整可。変換後の合計サイズと残容量を表示してからアップロード
- **アニメ GIF**: 専用対応はしない。ブラウザ側で動画と同じ経路(canvas 描画)で MJPG 化できる場合は動画として扱う

### 3.4 ストレージ(LittleFS 約 5MB)

```text
/content/<id>.jpg        静止画(id はアップロード時に採番される 8 桁 hex)
/content/<id>.mjp        動画
/content/<id>.thm        サムネイル 80×80 JPEG(ブラウザ側で生成して同時アップロード)
/playlist.json           プレイリスト(順序・各項目の名前・種別・表示秒数)
/config.json             表示設定(明るさ・デフォルト表示秒数・動画ループ回数)
/wifi.json               STA 資格情報
```

- `playlist.json` の例:

```json
{
  "items": [
    { "id": "a1b2c3d4", "type": "image", "name": "logo", "duration_s": 5 },
    { "id": "e5f6a7b8", "type": "video", "name": "intro", "loops": 2 }
  ]
}
```

- 容量管理: アップロード前に Web UI が `/api/status` の空き容量を確認。端末側でも受信中に空きを監視し、不足時はアップロードを中止してエラーを返す(書きかけファイルは削除)

### 3.5 スライドショー再生(基本状態)

- 起動後、プレイリストを先頭から自動巡回
  - 静止画: 項目ごとの `duration_s`(デフォルト 5 秒、Web UI で変更可)表示
  - 動画: `loops` 回(デフォルト 1)ループ再生して次へ。プレイリストが動画 1 件のみの場合は無限ループ
- プレイリストが空の場合: 組み込みデフォルト画像(ファームウェア埋め込み)と「QR で画像を送ってね」の案内を表示
- 動画再生: MJPG フレームを逐次読み出し・JPEG デコードして描画。フレーム全体をメモリに展開しない。目標 10fps(実測で調整)

### 3.6 端末操作

| 入力 | 状態 | 動作 |
| --- | --- | --- |
| ダイヤル回転 | スライドショー中 | 次/前のコンテンツへ即切替。手動モードに入り、10 秒無操作で自動巡回に復帰 |
| ダイヤル回転 | QR モード | Wi-Fi QR ↔ URL QR 切替(AP 時) |
| タッチ(タップ) | スライドショー中 | 一時停止 ↔ 再開(一時停止中は小さな pause アイコン表示) |
| BtnA 長押し 2 秒 | スライドショー中 | QR モードへ |
| BtnA 短押し | QR モード | スライドショーへ戻る |
| BtnA 短押し | スライドショー中 | ステータス画面(モード / SSID / IP / 空き容量 / コンテンツ数)を 5 秒表示 |

- ダイヤル操作時はブザーで短いクリック音を鳴らす(デフォルト ON、config で無効化可)

### 3.7 Web UI

単一ページ(ファームウェアに gzip 圧縮で埋め込み)。スマホブラウザ前提のレスポンシブ。日本語 UI。

- **ホーム / プレイリスト**
  - 保存済みコンテンツをサムネイル一覧表示(名前・種別・サイズ)
  - ドラッグ(または上下ボタン)で並べ替え、スワイプ/ボタンで削除、静止画の表示秒数を項目ごとに設定
  - 「今すぐ表示」ボタン: 選択コンテンツへ即ジャンプ
  - 容量バー(使用量 / 空き)
- **アップロード**
  - ファイル選択(`accept="image/*,video/*"`、複数選択可)
  - 各ファイルのプレビュー(円形マスク付きで実際の見え方を確認)、中央クロップ位置の調整
  - 動画: fps / 品質 / 開始位置の調整、変換後サイズの見積もり表示
  - 変換 → アップロード(進捗バー表示)。完了後プレイリスト末尾に自動追加され、端末側は即座に新コンテンツを反映
- **設定**
  - デフォルト表示秒数、動画ループ回数、LCD 明るさ、ブザー有効/無効
  - Wi-Fi(STA)設定: SSID / パスワード入力 → 保存 → 再起動。周辺 SSID のスキャン結果から選択可
  - 「AP モードに戻す」(STA 資格情報の削除)

### 3.8 REST API

| エンドポイント | メソッド | 内容 |
| --- | --- | --- |
| `/` | GET | Web UI(gzip HTML) |
| `/api/status` | GET | fw バージョン、Wi-Fi モード / SSID / IP、空き容量、コンテンツ数 |
| `/api/playlist` | GET | プレイリスト取得 |
| `/api/playlist` | POST | プレイリスト更新(順序・duration・loops) |
| `/api/upload` | POST | コンテンツアップロード(multipart、チャンク処理)。query: `type=image\|video`、`name`。応答に採番 `id` |
| `/api/content/<id>` | DELETE | コンテンツ削除(実ファイル + プレイリストから除去) |
| `/api/thumb/<id>` | GET | サムネイル JPEG |
| `/api/show/<id>` | POST | 指定コンテンツへ即ジャンプ |
| `/api/config` | GET / POST | 表示設定の取得 / 保存 |
| `/api/wifi/scan` | GET | 周辺 SSID スキャン結果 |
| `/api/wifi` | POST | STA 資格情報保存 → 再起動 |
| `/api/wifi` | DELETE | STA 資格情報削除(AP モードへ)→ 再起動 |
| (接続性チェック URL 群) | GET | 案内ページへリダイレクト(AP 時のみ) |

- 認証はなし(AP はオープンだが到達範囲が限定的、STA は家庭内 LAN 前提)。PIN 保護は v1 では実装しない(決定済み)

### 3.9 LCD 画面一覧

| 画面 | 内容 |
| --- | --- |
| スライドショー | コンテンツ全画面表示(基本状態) |
| 空状態 | 組み込み画像 + 「BtnA 長押しで QR」案内 |
| QR(Wi-Fi) | Wi-Fi 参加 QR + 手順 1/2 |
| QR(URL) | URL QR + 手順 2/2 |
| ステータス | モード / SSID / IP / 空き容量 / コンテンツ数 |
| アップロード中 | 進捗表示(受信バイト数 / 進捗バー) |
| エラー | エラー内容(容量不足・STA 接続失敗など)を数秒表示 |

## 4. 非機能要件

| 項目 | 要件 |
| --- | --- |
| 動画フレームレート | 10fps を目標(下限 8fps)。達成不能な場合は仕様の fps 既定値を下げる |
| 静止画切替 | 表示更新 500ms 以内 |
| アップロード速度 | 実効 100KB/s 以上を目標(SoftAP 環境) |
| 同時接続 | Web UI 利用は 1 クライアント想定(排他制御は行わないが、アップロードの同時実行は 409 で拒否) |
| 安定性 | 24 時間連続再生でクラッシュ・メモリリークなし |
| 再起動時間 | 電源投入からスライドショー開始まで 5 秒以内(STA 試行時は +15 秒) |

## 5. メモリ・フラッシュ設計指針

- パーティション(8MB): app 3MB / LittleFS 約 4.9MB / NVS ほか(OTA なし)
- アップロード受信バッファ: 4KB チャンク処理(splatoon 踏襲)
- 動画再生: フレーム単位で LittleFS → SRAM(最大フレームサイズ 48KB を上限としバッファ確保)→ JPEG デコード描画。ダブルバッファはメモリと相談(P4 で実測判断)
- Web UI HTML: gzip 圧縮で PROGMEM 埋め込み(目標 60KB 以下)
- BLE は使用しない(RAM 節約)

## 6. 要確認事項

すべて確認済み(2026-07-09)。

1. AP はパスワードなしのオープン AP とする(1 タップ接続を優先、リスクは用途上許容)
2. ブザー音: あり(デフォルト ON、設定で OFF 可)
3. STA モード時の PIN 保護: v1 では実装しない
4. 組み込みデフォルト画像: 仮のサンプル画像を用意(`assets/builtin.png`)。後日差し替え可
